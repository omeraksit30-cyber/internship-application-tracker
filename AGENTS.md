# Project Purpose

- This is a learning and portfolio project for a third-year Software Engineering student.
- The goal is to learn Java, Spring Boot, REST APIs, databases, testing and Git through practice.

# Technology Rules

- Use Java 21.
- Use Spring Boot 4.1.0.
- Use Maven Wrapper commands.
- Use `jakarta.persistence` and `jakarta.validation` packages.
- Do not use Lombok.
- Do not add a new dependency without explaining why and asking for approval.
- Do not put secrets, passwords, API keys or real personal data in the repository.

# Development Rules

- Implement only one small feature at a time.
- Explain every new class, annotation and method in Turkish.
- Do not create unnecessary abstractions.
- Keep the code beginner-friendly but professionally structured.
- Do not implement controller, service or repository unless explicitly requested.
- Run `./mvnw test` after every code change.
- If a test fails, explain the cause before attempting a fix.
- Do not commit or push automatically.
- Stop after completing the requested task and wait for approval.

# Git Rules

- Keep commits small and meaningful.
- Never add `target/`, `.DS_Store`, IDE folders or secret files.
- Suggest a commit message after each completed stage.
